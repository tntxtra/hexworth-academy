Security Best Practices Overview

Welcome to the comprehensive CompTIA A+ Core 2 Security Best Practices training module. This interactive tool covers all essential security concepts you need to master for the A+ certification and real-world IT security implementation.

Physical Security

  • Door locks and biometric access
  • Mantraps and access control vestibules
  • Security cameras and surveillance
  • Access badges and key fobs
  • Cable locks for devices
  • Server room security

Logical Security

  • Strong password policies (8+ chars, complexity)
  • Multi-Factor Authentication (MFA/2FA)
  • Principle of least privilege
  • User Account Control (UAC)
  • Admin vs. standard accounts
  • BitLocker and EFS encryption

Network Security

  • Windows Firewall configuration
  • VPN connections (IPSec, SSL/TLS)
  • Secure protocols (HTTPS, SSH, SFTP)
  • WPA3 wireless encryption
  • Disable SSID broadcast
  • MAC address filtering

Data Security

  • Secure data destruction (wipe, degauss, shred)
  • Data classification levels
  • 3-2-1 backup rule
  • Encryption at rest and in transit
  • Data Loss Prevention (DLP)
  • Secure disposal methods

Social Engineering Awareness

  • Phishing and spear phishing
  • Vishing (voice phishing)
  • Tailgating and piggybacking
  • Shoulder surfing prevention
  • Impersonation tactics
  • Security awareness training

Mobile Device Security

  • Mobile Device Management (MDM)
  • Screen locks (PIN, pattern, biometric)
  • Remote wipe capabilities
  • Device encryption
  • App security and permissions
  • BYOD policies

Browser Security

  • Pop-up blockers
  • Private browsing mode
  • Clear cache and cookies
  • Certificate warnings
  • Browser extensions security
  • Password manager integration

Authentication Methods

  • Something you know (password)
  • Something you have (token, smartcard)
  • Something you are (biometrics)
  • Single Sign-On (SSO)
  • RADIUS and TACACS+
  • Kerberos authentication

Key Security Principles

Defense in Depth

Implement multiple layers of security controls to protect assets. If one layer fails, others continue to provide protection.

Least Privilege

Users should have only the minimum access rights necessary to perform their job functions. Reduces potential damage from compromised accounts.

Security Through Obscurity (Not Recommended Alone)

Hiding system details can provide minimal protection but should never be the primary security mechanism.

AAA Framework

Authentication: Verify identity | Authorization: Grant appropriate access | Accounting: Track user actions

Security Risk Assessment

Analyze real-world scenarios and identify security vulnerabilities. Select the primary security risk for each scenario.

Scenario 1: Unsecured Workstation

An employee leaves their workstation unlocked when going to lunch. Their computer displays sensitive customer data on the screen, and the office has visitors walking through the area.
A. Physical security - Unauthorized physical access to data
B. Network security - Firewall misconfiguration
C. Malware infection from unlocked computer
D. Weak password policy

Scenario 2: Email From "IT Department"

An employee receives an email claiming to be from the IT department, requesting they click a link to verify their account credentials due to a "security update." The email has poor grammar and the sender's address is from a free email service.
A. Outdated antivirus software
B. Social engineering - Phishing attack
C. Lack of firewall protection
D. Missing BitLocker encryption

Scenario 3: Wireless Network Configuration

A small business is using WEP encryption on their wireless network with the default admin password on the router. The SSID is broadcasting as "CompanyName-WiFi" and MAC filtering is not enabled.
A. Social engineering vulnerability
B. Outdated operating systems
C. Network security - Weak wireless encryption (WEP) and default credentials
D. Missing antivirus protection

Scenario 4: Laptop Disposal

A company is upgrading laptops and donating the old ones to a local school. The IT department performs a "quick format" on each laptop before donation but does not use any additional data destruction methods.
A. Firewall not configured properly
B. Data security - Inadequate data destruction (recoverable data)
C. Missing MFA implementation
D. Weak password policy

Scenario 5: Remote Worker Access

A remote employee connects to the company network by opening port 3389 (RDP) directly to their office workstation through the firewall. They use a simple 6-character password with no additional authentication required.
A. Physical security breach
B. Network security - Direct RDP exposure with weak authentication (no VPN/MFA)
C. Social engineering attack
D. Malware infection only

Test your knowledge with these 10 CompTIA A+ Core 2 security questions.

Question 1: Which of the following is the BEST example of multi-factor authentication?

A. Password and security question
B. Password and fingerprint scan
C. PIN and username
D. Two different passwords
Correct Answer: B
Multi-factor authentication requires two different types of authentication factors: something you know (password), something you have (token/card), or something you are (biometric). Password + fingerprint combines "know" and "are" factors.

Question 2: A company wants to ensure data on decommissioned hard drives cannot be recovered. Which method is MOST effective?

A. Quick format
B. Delete all files and empty recycle bin
C. Physical shredding or degaussing
D. Reinstall the operating system
Correct Answer: C
Physical shredding destroys the drive mechanically, while degaussing scrambles magnetic data. Both make data completely unrecoverable. Software methods (format, delete, reinstall) leave data recoverable with forensic tools.

Question 3: Which wireless encryption standard provides the STRONGEST security?

A. WEP
B. WPA
C. WPA2
D. WPA3
Correct Answer: D
WPA3 is the newest and most secure wireless encryption standard. It uses stronger encryption (SAE instead of PSK), protects against brute force attacks, and provides better security even with weak passwords. WEP is completely broken, WPA is deprecated, and WPA2 is vulnerable to KRACK attacks.

Question 4: An attacker calls an employee pretending to be from IT support and asks for their password. This is an example of:

A. Phishing
B. Vishing
C. Tailgating
D. Shoulder surfing
Correct Answer: B
Vishing (voice phishing) is social engineering via phone calls. Phishing uses email, tailgating is physical following, and shoulder surfing is observing someone entering credentials.

Question 5: Which principle states that users should only have the minimum access necessary to perform their job?

A. Defense in depth
B. Principle of least privilege
C. Separation of duties
D. Security through obscurity
Correct Answer: B
The principle of least privilege limits user access rights to only what's necessary for their role. This minimizes potential damage from compromised accounts or insider threats.

Question 6: What does BitLocker provide for Windows systems?

A. Antivirus protection
B. Full disk encryption
C. Firewall protection
D. Spam filtering
Correct Answer: B
BitLocker provides full disk encryption for Windows systems, protecting data if a device is lost or stolen. It can use TPM (Trusted Platform Module) for enhanced security.

Question 7: What is the 3-2-1 backup rule?

A. 3 backups per day, 2 different times, 1 verification
B. 3 copies of data, 2 different media types, 1 offsite
C. 3 servers, 2 locations, 1 administrator
D. 3 passwords, 2 factors, 1 token
Correct Answer: B
The 3-2-1 rule is a backup best practice: maintain 3 copies of your data (1 primary + 2 backups), store backups on 2 different media types (e.g., disk and tape), and keep 1 copy offsite for disaster recovery.

Question 8: Which feature in Windows prompts users when administrative actions are attempted?

A. Windows Defender
B. User Account Control (UAC)
C. Windows Firewall
D. BitLocker
Correct Answer: B
User Account Control (UAC) prompts users for permission or administrator credentials when programs try to make system changes. This prevents unauthorized modifications and helps protect against malware.

Question 9: A security guard physically walks through a space to verify everyone has proper authorization. This is an example of:

A. Logical security
B. Physical security
C. Network security
D. Data security
Correct Answer: B
Physical security involves tangible measures to protect facilities, equipment, and personnel. Security guards, locks, cameras, and mantraps are all physical security controls.

Question 10: Which mobile device management feature allows IT to erase all data remotely?

A. Screen lock
B. GPS tracking
C. Remote wipe
D. App whitelisting
Correct Answer: C
Remote wipe allows administrators to erase all data on a mobile device remotely, protecting company information if a device is lost or stolen. This is a critical MDM feature.
