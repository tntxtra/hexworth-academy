Browser Security Checklist

0%
Complete these essential steps to harden your browser security. Click each item to mark it as complete and track your progress.

Enable HTTPS-Only Mode

Force all connections to use encrypted HTTPS protocol. This prevents your data from being transmitted in plain text and protects against man-in-the-middle attacks.

Block Third-Party Cookies

Prevent advertisers and trackers from following you across the web. Third-party cookies enable cross-site tracking and compromise your privacy.

Disable JavaScript on Sensitive Sites

Use extensions like NoScript to control JavaScript execution. While this may break some functionality, it significantly reduces attack surface on high-risk sites.

Install Content Blockers

Use uBlock Origin or similar tools to block malicious scripts, ads, and tracking pixels. These tools prevent many common web-based attacks before they reach your browser.

Clear Browsing Data Regularly

Schedule regular clearing of cookies, cache, and browsing history. This limits the data available to attackers and reduces your digital footprint.

Verify Site Certificates

Always check the padlock icon before entering sensitive information. Click it to view certificate details and ensure the connection is legitimately encrypted.

Disable Browser Password Autofill

Turn off autofill for passwords and use a dedicated password manager instead. Browser autofill is vulnerable to form-grabbing attacks.

Enable Enhanced Tracking Protection

Activate your browser's built-in tracking protection (Strict mode recommended). This blocks known trackers, cryptominers, and fingerprinting scripts.

Keep Browser Updated

Enable automatic updates and ensure you're running the latest version. Security patches are critical for protecting against newly discovered vulnerabilities.

Use DNS-over-HTTPS (DoH)

Encrypt your DNS queries to prevent ISPs and attackers from seeing which sites you visit. Configure DoH in your browser's network settings.

Disable WebRTC

WebRTC can leak your real IP address even when using a VPN. Disable it if you don't use video calling features in your browser.

Review and Remove Unnecessary Extensions

Audit your extensions monthly. Each extension increases attack surface and may have access to sensitive data. Only keep what you actively use.

Common Certificate Errors:
  • NET::ERR_CERT_DATE_INVALID - Certificate has expired or is not yet valid
  • NET::ERR_CERT_COMMON_NAME_INVALID - Certificate domain doesn't match the website
  • NET::ERR_CERT_AUTHORITY_INVALID - Certificate issued by untrusted authority
  • NET::ERR_CERT_REVOKED - Certificate has been revoked by the issuer
Best Practices:
  • Never proceed past certificate warnings on financial or sensitive sites
  • Click the padlock icon to view certificate details before entering credentials
  • Look for Extended Validation (EV) certificates on banking sites
  • Be wary of certificate errors on familiar sites (possible MITM attack)

Password Manager Guide

Why Use a Password Manager?
Password managers generate, store, and autofill strong unique passwords for every account. This eliminates password reuse (the #1 cause of account compromises) and makes it practical to use complex passwords you can't remember.

Master Password Best Practices

Your master password is the key to everything. Follow these rules:
  • Length over complexity: Use 5-7 random words (passphrase) rather than complex gibberish
  • Example: "correct horse battery staple autumn" (5 words = ~65 bits of entropy)
  • Never reuse: Your master password should be unique and never used anywhere else
  • Memorize it: Never write it down digitally. Practice typing it until it's muscle memory
  • Enable 2FA: Add two-factor authentication to your password manager account
  • Set up recovery: Configure emergency access or recovery codes (store offline)
Migration Tips: Moving to a password manager is easier than you think. Start by importing existing passwords, then gradually update them to strong unique passwords. Use the password generator for all new accounts. Within a month, you'll wonder how you ever managed without it.

Universal Privacy Recommendations

Regardless of browser, always:
  • Use private/incognito mode for sensitive browsing (banking on public computers)
  • Regularly clear cookies and site data
  • Review and revoke unnecessary site permissions monthly
  • Disable autofill for forms and passwords (use a password manager instead)
  • Consider using containers or profiles to separate different browsing contexts
  • Install privacy-focused extensions (uBlock Origin, Privacy Badger)

Common Browser Threats

Understanding threats is the first step in defending against them. Learn to recognize and protect against these common browser-based attacks.

Phishing Attacks

CRITICAL

What it is: Fraudulent websites that mimic legitimate sites to steal credentials.

How it works: Attackers send emails or messages with links to fake login pages that look identical to real services.

Defense:

  • Always check URL before entering credentials
  • Look for HTTPS and valid certificates
  • Never click links in suspicious emails
  • Use password manager (won't autofill on wrong domain)
  • Enable 2FA on all accounts
üì•

Drive-by Downloads

HIGH

What it is: Malware automatically downloaded to your system without your knowledge or consent.

How it works: Compromised or malicious websites exploit browser vulnerabilities to install software silently.

Defense:

  • Keep browser and plugins updated
  • Use script blockers (uBlock Origin, NoScript)
  • Disable auto-download features
  • Avoid untrusted or sketchy websites
  • Run antivirus with real-time protection
üíâ

Cross-Site Scripting (XSS)

HIGH

What it is: Injection of malicious scripts into trusted websites viewed by other users.

How it works: Attackers inject JavaScript into web pages that executes in victims' browsers, stealing data or hijacking sessions.

Defense:

  • Enable Content Security Policy (CSP) if available
  • Use NoScript or similar extensions
  • Keep browser updated (XSS filters built-in)
  • Log out of sensitive sites when done
  • Use browser extensions cautiously
üïµÔ∏è

Man-in-the-Middle (MITM)

CRITICAL

What it is: Interception of communications between your browser and websites you visit.

How it works: Attackers position themselves between you and the server, reading or modifying traffic (especially on public WiFi).

Defense:

  • Only use HTTPS (enable HTTPS-Only mode)
  • Never ignore certificate warnings
  • Avoid public WiFi for sensitive activities
  • Use VPN on untrusted networks
  • Install HTTPS Everywhere extension
üç™

Cookie/Session Hijacking

HIGH

What it is: Theft of session cookies to impersonate you on websites where you're logged in.

How it works: Attackers steal your session cookies through XSS, network sniffing, or malware and use them to access your accounts.

Defense:

  • Log out of accounts when finished
  • Use HTTPS everywhere
  • Clear cookies regularly
  • Enable cookie protection in browser
  • Use separate browsers for sensitive tasks

Browser Fingerprinting

MEDIUM

What it is: Tracking technique that identifies you based on unique browser configuration without cookies.

How it works: Websites collect information about your browser, OS, plugins, fonts, and settings to create a unique "fingerprint."

Defense:

  • Use privacy-focused browsers (Firefox, Brave)
  • Minimize installed extensions
  • Disable WebGL and Canvas fingerprinting
  • Use fingerprint protection extensions
  • Regularly change browser configurations
Layered Defense is Key: No single security measure is perfect. Combine multiple defenses (updated browser + extensions + good habits + 2FA + password manager) to create robust protection against browser-based threats.

Browser Extension Security

Extensions are powerful but risky. Every extension you install can access your browsing data, modify web pages, and potentially compromise your security. Choose carefully and audit regularly.

Safe Extension Practices

Research Before Installing

Check the developer, read reviews, verify permissions, and search for any security incidents. Look for extensions with many users and positive ratings over time.

Review Permissions Carefully

Extensions requesting "Read and change all your data on all websites" have complete access to everything you do. Only grant broad permissions if absolutely necessary.

Minimize Extension Count

Each extension increases attack surface and performance overhead. Audit monthly and remove anything you don't actively use. Aim for fewer than 5 extensions.

Prefer Open Source Extensions

Open source extensions can be audited by security researchers. Examples: uBlock Origin, Privacy Badger, HTTPS Everywhere, Bitwarden.

Keep Extensions Updated

Enable automatic updates and check periodically. Outdated extensions may contain known vulnerabilities that attackers can exploit.

Avoid Extension Clones

Malicious developers create fake versions of popular extensions. Always install from official sources and verify the developer name matches the legitimate version.

Monitor Extension Behavior

Watch for unusual activity: unexpected ads, redirects, slow performance, or modified search results. These may indicate a compromised or malicious extension.

Use Site-Specific Permissions

When possible, restrict extensions to specific sites rather than granting access to all websites. Right-click the extension icon to manage site access.
